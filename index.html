What I can do for you

  • I mostly focus on Django framework and can build new project for you using Django or improve existing one. I don't mind to work on someone else's code.
  • I can also do anything related to JavaScript. jQuery is my tool of choice but I have some experience with different frameworks like backbone.js . I'm familiar with advanced javascript concepts like closures or modules as described in Douglas Crockford's book: "JavaScript: The Good Parts". By the way, really great book.
  • I'm also familiar with HTML/CSS and can do simple fixes if needed or handle CSS framework like Twitter's Bootstrap. But please, do not expect me to turn psd files into working HTML/CSS layout.
  • I usually manage Linux servers for my clients so I should have no problems dealing with your setup whatever it is. So far I managed WebFaction and Linode accounts but I can deal with any Linux based VPS or dedicated server. I use fabric for automatic deployment.
  • I know my way around relational databases, especially PostgreSQL and MySQL. I can install database on your server, setup backups, design tables layout, preform data migration using South if needed.
  • I use version control systems as one of my basic tools. I mostly use Git these days but I'm also familiar with Baazar (bzr), Subversion (svn) and Mercurial (hg).
But these are just technical skills - I'm pretty sure all other software developers have them too. But with time I also discovered that there is more than that.
  • I'm reliable: I keep my deadlines and don't disappear in the middle of the project.
  • I'm independent and can work with no supervision so you won't need to hold my hand all the time ;) Actually, I hate being micromanaged and fire customers that try to do that.
  • I can not only code but also give a lot of helpful suggestions.
  • I have experience with bigger projects and know how to build clean and readable code.
  • And of course I have very high attention to detail - I won't neglect privileges check or more complex validation rules.
  • Not to mention my communications skills. Nothing more annoying than having 3 iterations just to 'force' the developer to follow the instructions to the letter - it's like they don't really read them ;) I also use bullet points a lot, try to write about one thing per email, love sending close-ended emails to save my client's time.
  • I'm willing to learn new things. Web development is very broad subject with plenty of different tools. It's not possible to know everything so learning new things is very important for us, developers.
  • I'm also don't mind to wear many hats. Most of my clients are individuals or small companies and hiring a backend developer, database administrator and server administrator is usually an overkill. I can do all of that myself. I must admit I'm probably not a great server or database administrator, but I'm good enough to keep things up and running before your project grow big enough to extend your team.

At the end I will just say that so far all my previous clients were very happy so I probably know what am I doing ;)
