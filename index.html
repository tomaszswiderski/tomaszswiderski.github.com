What I worked on so far

Grono.net

Grono.net was a large social networking website in Poland. Apart from hosting many discussion forums, it contained features allowing the sharing of photos, links to cultural events in particular cities, the sale of property and job searches. Although the site itself was free, there were a number of additional features for paying subscribers, such as the ability to moderate your own forums, take part in competitions, upload more photos and remove on-site advertisements. It required an invitation from an existing member to register. It had over 2 million members. My role in this project included:

  • implementing in-browser instant messaging. It was simple Twisted server as backend and COMET server (Orbited) as middleware between Twisted and browser.
    • implementing AJAX-based commenting system.
  • technical assistance during few marketing campains launches, for example: handling communication between flash game (marketing campain) and site's servers in secure way.
  • and of course lots of other smaller changes.

JKI Data Manager

It's restaurants supplies management system built exclusively on highly customized Django admin site. I worked on this project alone from very beginning. This project covered:

  • Adding custom Javascript widgets like price graphs or handling simple interactins with lightbox instead of page reload.
  • Importing data from csv and xls files.
  • Detecting price conflicts in imported and entered data.
  • Automatic email remainders about expiring agreements.
  • Custom changelist filters.
  • And lots of other changes.

Icing on the cake

Online shop that belong to bakery located in New Zealand. I took over this project after other software develeper.

  • I help them to fix their credit card processing application that sometimes charged their cliens multiple times for the same purchase. They use unique New Zealand's payment processor: DPS, so it was a bit refreshing after working with PayPal API multiple times in a row ;)
  • As always I had a change to work with Javascript too. I added timezone support to their order form calendar so all order validation rules could be handled in local New Zealand time.
  • I also done numerous minor fixed to their Django backend.

TradingSystem

Stock market backtesting system I was working on in it's early stage. Beside working with standard Django framework features, I had an opportunity to:

  • work with jQuery charting libraries.
  • try PyTables and NumPy library for handling large amounts of data.

Evivia

Website application that allowed to create scheduled marketing campains on social media sites like Facebook, Twitter or even RSS feed. I worked on many aspect of their aplication:

  • I worked on Twitter/Facebook integration - on both posting and pulling posts from those sites using their APIs.
  • Their scheduling mechanism - at the time I worked on this project it was based on custom Django management commands executed by cron tasks. Now I would use Celery for this problem.
  • I added few custom jQuery scripts to this project, all of them handled AJAX requests.
  • And other standard Django things like create/read/update/delete views.

IDigYa

Funny dating-like application. I wasn't the original developer who created this website but I made few improvement requested by client:

  • shrinked whole website to fit it into 1024px browser window
  • implemented beautiful jQuery-powered tutorial for first time users
  • added few other improvements to their Javascript/jQuery layer
  • of course I worked with their Django source code too, my changes included:
    • improvements to Facebook signup process
    • improvements to their opt out mechanism for SMS notifications
    • ability to block other users messages
    • and many other minor changes
